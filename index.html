D ‚Äì Determine the Goals

Objective: To evaluate the usability, interface design, and user experience of a web-based mobile store system composed of:

  • Admin Panel ‚Äì for managing products, orders, and inventory
  • Client Panel ‚Äì for browsing products, managing cart, and placing orders

Key Goals:

  • Allow users to register, log in, and manage their accounts easily
  • Ensure smooth product browsing, search, and filtering
  • Test the shopping cart, checkout flow, and PayPal integration
  • Assess admin functionalities like product and order management
  • Identify usability problems and recommend UI/UX improvements
  • Enhance feedback, task flow, and error handling

E ‚Äì Explore the Questions

For Clients:

  • Can users register and log in without issues?
  • Is the homepage clear, appealing, and informative?
  • Do filters and brand checkboxes work properly?
  • Is the ‚ÄúAdd to Cart‚Äù and checkout process easy to follow?
  • Are error messages visible and understandable?
  • Can users view orders and update their info smoothly?

For Admins:

  • Is the admin login secure and functional?
  • Can admins easily add/edit brands, categories, and products?
  • Is managing orders and banners clear and efficient?
  • Do confirmation prompts appear when needed (e.g., delete/edit)?
  • Are labels understandable even for non-technical users?

C ‚Äì Choose the Evaluation Approach and Methods

Methods Used:

  • Heuristic Evaluation (Nielsen‚Äôs 10 Usability Heuristics)
  • User Testing (client and admin users performing tasks)
  • Observation and Note-Taking (tracking errors and struggles)
  • Group Collaboration (combining feedback from all members)

Tools Used:

  • Google Docs ‚Äì for documenting usability issues
  • Browser Developer Tools ‚Äì for inspecting UI elements
  • Google Forms ‚Äì optional user feedback collection

Expected Outcomes:

  • Identify UI/UX issues like unclear labels and missing feedback
  • Understand user pain points and confusion
  • Generate recommendations and an action plan

  • Users: 5‚Äì7 testers (client) and 2‚Äì3 evaluators (admin)

  • No Undo/Redo options
  • Missing help tooltips or on-screen hints

Evaluation Results

Here are the key usability issues found during the evaluation, along with the violated heuristic, affected users, and recommended improvements.

Lack of Input Validation

Heuristic Violated: Error prevention

Description: Users could submit forms with missing or incorrect data (e.g., empty fields, invalid email format).

Affected Users: Clients during registration or checkout

Recommendation: Implement input validation and show real-time error messages for required fields and formats.

Slow Load Times on Some Pages

Heuristic Violated: Visibility of system status

Description: Pages like ‚ÄúOrders‚Äù or ‚ÄúProducts‚Äù in the admin panel took a few seconds to load without any feedback.

Affected Users: Admins

Recommendation: Add loading indicators (spinners or messages) while data is being fetched to reduce user confusion.

Navigation Buttons Too Small on Mobile

Heuristic Violated: Flexibility and efficiency of use

Description: Some clickable icons or buttons (e.g., cart, login) are too small and hard to tap on mobile devices.

Affected Users: Mobile users

Recommendation: Increase button size and spacing for better mobile accessibility, especially for touch screens.

Poor Mobile Responsiveness

Heuristic Violated: Flexibility and efficiency of use

Description: Elements overlapped or broke layout on smaller screens.

Affected Users: Clients accessing via mobile

Recommendation: Implement responsive CSS frameworks (e.g., Bootstrap or Tailwind) and test across devices.

Confusing Technical Terminology

Heuristic Violated: Match between system and real world

Description: Labels such as ‚Äúslug,‚Äù ‚Äústatus,‚Äù and ‚Äúbanner‚Äù confused non-technical users during admin use.

Affected Users: Admins (non-technical)

Recommendation: Use more user-friendly terms or add tooltips for clarification.

Proposed Improvements

Based on the evaluation results, here are the proposed improvements with justifications grounded in HCI principles and Nielsen‚Äôs heuristics.

Input Validation with Real-Time Feedback

Improvement: Add real-time form validation and error prompts for fields like email, password, and name.

Justification: Addresses Error Prevention by reducing invalid submissions and user frustration.

Loading Indicators for Slow Pages

Improvement: Display loaders or spinners while waiting for content to load (e.g., Orders or Products page).

Justification: Enhances Visibility of System Status and keeps users informed.

Mobile-Friendly Button Sizes

Improvement: Enlarge small buttons (e.g., cart, login) and ensure enough spacing for touch interaction.

Justification: Improves Flexibility and Efficiency of Use especially on mobile devices.

Fully Responsive Design

Improvement: Use responsive CSS frameworks or media queries to fix broken layouts on smaller screens.

Justification: Ensures usability for various screen sizes and supports responsive access.

Replace Technical Jargon

Improvement: Change terms like ‚Äúslug‚Äù or ‚Äústatus‚Äù with simpler labels or add explanatory tooltips.

Justification: Supports Match Between System and Real World for better user comprehension.
